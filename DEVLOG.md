# 開発日誌 (Dev Log)

## 2026-02-25: モデルの選定とレイテンシ改善
**【課題】**
生成AIを活用した機能（メニュー画像の解析）において、APIの応答待ち時間がUXに悪影響を与えていた。

**【検証】**
メニュー画像 (`menuphoto/menuphoto.jpeg`) を使用し、ローカルで各種Geminiモデルの応答速度のパフォーマンステストを実施した。

**テスト結果:**
- `gemini-2.0-flash` : 約 11.7秒 ~ 12.2秒
- `gemini-2.5-flash-lite` : **約 9.7秒**
- `gemini-2.5-flash` : 約 22.7秒
- `gemini-3-flash-preview` : 約 25.0秒

**【結論と対応】**
最新の軽量モデルである `gemini-2.5-flash-lite` が最も応答速度が速く、かつ当該タスク（OCR、構造化、翻訳、英語クエリの生成）において十分な精度を出せることを確認した。
そのため、`api/analyze/route.ts` 内で使用するモデルを `gemini-2.0-flash` から `gemini-2.5-flash-lite` に変更した。

**【📌 今後の開発者へ】**
UI/UXの観点から「待ち時間の短縮」は当アプリの最重要課題の一つです。
今後、精度向上のためにより大きなモデル（`Pro`系や重い`Flash`系）に変更したくなる誘惑に駆られるかもしれませんが、それは**「待ち時間の増大（ユーザーの離脱）」とトレードオフ**になります。
画像解析を伴うためAPIコールが重くなりがちですが、基本の推論エンジンは常に**最新の「Lite」系または最速のモデル**を採用し、精度はプロンプトエンジニアリングやフロントエンド側のUX（プレースホルダーや非同期処理）でカバーしてください。無闇に古い重いモデルに戻さないように注意してください。

## 2026-02-25: UX改善版のVercelデプロイ (v0.1.x)
**【概要】**
「待ち時間を感じさせないUX」をテーマにした以下の改善を `main` ブランチにマージし、本番環境（Vercel）へデプロイを実施した。

**【主な変更点】**
1. **Geminiモデルの刷新:**
   - 応答速度を最優先し、解析モデルを `gemini-2.0-flash` から `gemini-2.5-flash-lite` に変更。（約12.2秒 → 9.7秒への短縮を確認）
2. **地域Tipsのストリーミング表示:**
   - OpenAI SDK (Gemini API) の `generateContentStream` を利用し、サーバー側からストリーミング応答を実装。
   - フロントエンドでチャンクをパースし、JSON全体の完成を待たずに `"detected_country_code"` を最速で抽出。
   - 抽出した国コードに基づいて、現地の食文化Tips（チップ相場など）をローディング画面に動的表示させることで、待機時間を「知識を得る時間」に昇華させた。
3. **Hero画像（テーブルスプレッド）のSkeleton実装:**
   - メニューテキストが最速で表示された後、テーブル画像が生成完了するまでの間、高品質なSkeleton UI（フワフワ光るアニメーション）を表示。
   - 画像生成完了後、`animate-fade-in` で自然に差し替わるように実装。
4. **PM向け資料の追加:**
   - `pm_summary.md` (機能概要、ロードマップ、課金化戦略)
   - `DEVLOG.md` (本ファイル: 開発における重要な意思決定の記録)

**【デプロイフロー】**
- ローカルにて `npm run build` を実行し、静的/動的ルーティングのエラーがないことを確認。
- Githubへプッシュ後、VercelのAuto Deploymentがトリガーされ正常に完了。

## 2026-02-25: 国選択UIとTipsの多言語対応 (v0.2.x)
**【概要】**
さらなるUX向上と解析スピード（レイテンシ）の短縮のため、ユーザーによる「メニュの地域事前選択機能」と「現地の食文化Tipsの翻訳先言語（ローカライズ）対応」を `main` へコミットおよびデプロイした。

**【主な変更点】**
1. **メニュー地域の事前選択 UI の実装 (`page.tsx`)**:
   - メニュー画像のアップロード前に「Where are you eating? (対象国)」をユーザーが選択できるUIを追加した。
   - 初期値は `Auto-Detect` だが、手動で指定した場合、APIにその国情報が渡る。
2. **AI推論のショートカット (Fast-tracking) (`api/analyze/route.ts`)**:
   - 指定された対象国（例: "FR"）がある場合、Geminiのプロンプトに「このメニューはフランスのものです」といった強力なコンテキストを事前に注入。これにより言語や料理ジャンルの推論ステップが省略され、解析速度のさらなる底上げと翻訳精度の向上が見込める。
3. **Tipsのローカライズ表示 (`page.tsx`)**:
   - 既存の英語のみだったTipsを、ユーザーが選択している翻訳先言語 (`targetLang`) に合わせて表示する多言語辞書を構築。
   - `Auto-Detect` 以外が選ばれている場合、画像の解析開始ボタンを押した**瞬間に**（ストリーミングを待たずに）、そのターゲット言語で対象国のTipsが表示される最強のUXを実現。

## 2026-02-25: メニュー説明の高度化 (v0.3.x)
**【概要】**
「見出し＝説明」となってしまうことが多いという課題に対し、プロンプトエンジニアリングによって「未知の固有名詞や調理法を補足する、具体的で短い説明」を生成するように修正し `main` へコミットおよびデプロイした。

**【主な変更点】**
1. **Gemini プロンプトのチューニング (`api/analyze/route.ts`)**:
   - 単なる「簡単な説明(brief description)」という指示を強固な制約に書き換え。
   - 禁止事項: 翻訳した料理名をそのまま繰り返すな。
   - 推奨事項: 元の料理名に見慣れない地域の調理法や固有の食材（例：ガリシア風、グアンチャーレ）が含まれている場合、それが何でありどんな味かを簡潔に補足せよ。
   - 制約事項: コンパクトなUIに収まるよう、最大1〜2文の短い文章に制限せよ。
   - この指示により、ユーザー（旅行者）が直感的に料理のイメージを掴める最強の「専属ガイド」へと進化した。

## 2026-02-25: 待機画面のTipsローテーション機能 (v0.4.x)
**【概要】**
ローディング待機時間をさらに「楽しく知識を得るポジティブな時間」にするため、単一だったTipsを数秒ごとに切り替えるローテーションアニメーションを実装し、`main` へコミットおよびデプロイした。

**【主な変更点】**
1. **フロントエンド主導のハードコードTips (`page.tsx`)**:
   - 毎回Geminiで5個生成する案も見討されたが、「生成のAPIコール完了を待つ時間」が発生し、**ボタンを押した瞬間の爆速UX**が損なわれるため却下。
   - 代わりに、主要な国（JP, IT, ES, FR, US, KR, TH, TW）および各言語に対し、質の高い豆知識を**3つの配列データ**として事前にハードコードした。これによりAPIコールのレイテンシはゼロを維持している。
2. **ローテーションアニメーションの実装 (`page.tsx`)**:
   - `useEffect` と `setInterval` を用い、解析中 (`analyzing === true`) は3.5秒ごとに `tipIndex` をカウントアップ。
   - DOMの `key` 属性に `tipIndex` を付与することで、切り替わりのたびに `animate-fade-in` クラスが再評価され、フワッとオシャレに次のTipsが表示される高品質なUIを実現した。

## 2026-02-25: ネットワーク不安定時のリカバリーUX向上 (v0.5.x)
**【概要】**
旅行先（地下のレストランや海外ローミング環境など）での不安定なネットワーク接続を想定し、「どんなエラーが起きたのかユーザーが納得できる明確なメッセージ」の表示と、「通信が途絶えても、そこまで受信した不完全なテキストだけでメニューを表示する」というフォールバック（リカバリ）処理を実装した。

**【主な変更点】**
1. **JSON修復エンジンの搭載 (`page.tsx`)**:
   - Geminiからのストリーミング応答がネットワークの切断等で「ぶつ切り」になった場合、不完全なJSON文字列の末尾を手動で閉じ（`"]}` などの補完）、画面クラッシュ（`Unterminated string` 等のエラー）を防ぐ強力な防御ロジックを追加。
   - これにより、例えば「前菜とメイン」までしか翻訳できていないタイミングで通信が切れても、その時点までのメニューをユーザーに返すことができるようになった。
2. **エラーメッセージの人間化 (Human-readable Errors) (`page.tsx`)**:
   - 機内モード等の完全なオフライン時は即座にAPIコールを中断し、「Internet connection lost.」と表示。
   - `AbortController` による45秒の明示的タイムアウト設定を導入し、無限ロードを防ぎつつ「Connection timed out.」とわかりやすく表示。
3. **バックエンドのEdgeランタイム化 (`api/analyze/route.ts`)**:
   - `export const runtime = "edge";` を明示的に指定し、Vercel上でのストリーミング応答の安定性を向上させた。
