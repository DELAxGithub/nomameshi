# プロダクト概要：menumenu (メニュー翻訳＆視覚化アプリ)

## 📌 どんなアプリか（Core Value Proposition）
**「メニューを読むのではなく、料理の魅力を視覚的に味わう」** をコンセプトにした、AI駆動のメニュー翻訳・解析アプリです。
外国語のメニュー画像をアップロード・ペーストするだけで、単なる直訳ではなく、料理の構成やレストランの雰囲気を解析し、美しいUIで多言語化して表示します。

### 🌟 現在の主要機能（ソースコード解析より）
1. **メニューの解析・翻訳 (Gemini API利用)**
   - 画像（写真/スクリーンショット）からのOCR＆構造化
   - 対応言語: 日本語、英語、中国語、韓国語、スペイン語、フランス語
   - 単なる翻訳だけでなく、「レストランの雰囲気 (Vibe)」「料理の説明 (Description)」「価格」まで抽出
2. **テーブルイメージ（Hero画像）の自動生成**
   - 抽出した料理群（Image Query）をもとに、そのメニューの料理が並んだ「テーブルスプレッド画像」を生成し、視覚的に食欲をそそる設計
3. **共有・保存機能**
   - 翻訳フォーマットされたテキストのクリップボードコピー / ネイティブShare機能
   - 翻訳結果全体を美しいPNG画像として保存（html2canvasを利用）

## 📈 今後の育成方針（プロダクトロードマップ案）

外国での食事という「ペインが非常に強い体験」を解決するプロダクトであるため、旅行者（インバウンド・アウトバウンド両方）をコアターゲットに据えて育成します。

### 1. ユーザー体験（UX）の深化
- **アレルギー・食の嗜好対応:** ユーザー設定（ヴィーガン、グルテンフリー、甲殻類アレルギーなど）を事前に登録し、メニュー解析時に「食べられないもの」をハイライトして警告する機能。
- **パーソナライズされたメニュー履歴:** 過去に翻訳したメニューや、旅行先で食べたものを「食の旅の記録」として保存・振り返れる機能。
- **音声サポート:** 注文時に困らないよう、現地の言葉で「これをください」と発音する Text-to-Speech (読み上げ) 機能。

### 2. 解析・生成精度の向上
- 達筆な手書きメニューや、薄暗い店内での不鮮明な写真に最適化された画像前処理の導入。
- 料理画像の生成スピードと精度の更なる向上（料理固有のローカルな見た目の再現）。

### 3. 「待ち時間（レイテンシ）」を感じさせないUXの工夫（★重要）
Generative AIを使用する都合上、画像の解析とテーブル画像の生成にある程度の数秒〜十数秒の待ち時間が発生します。この「体感の待ち時間」を減らす工夫がリテンションに直結します。
- **スケルトンスクリーン＆プログレッシブローディング:** 「解析中…」という単一のスピナーではなく、取得できたメニューのセクション（前菜、メインなど）から順次画面に描画（ストリーミング表示）していく。
- **インタラクティブな待機画面:** ローディング中に「この地域の食文化の豆知識」や「現地のチップの相場」などをTipsとして表示し、待つ時間を「知識を得る時間」にすり替える。
- **Hero画像の非同期生成:** テキストの翻訳結果を先に最速で表示させ、重たいテーブル画像（Hero画像）はバックグラウンドで生成し、完了次第フワッと後読み（フェードイン）させる。（※現在はMenu表示後に非同期で画像を取得する実装になっていますが、Skeleton等でプレースホルダーを置くとより体感が良くなります）
- **モデルのチューニング:** スピードに特化した軽量モデル（Gemini Flask 8B等）の採用を検討する。

## 💰 課金化・マネタイズ戦略 (Monetization Strategy)

現在はシンプルなツールですが、ユーザーの利用文脈（旅行中、外食中＝財布の紐が緩みやすい）を活かし、B2CとB2Bの両面からアプローチ可能です。

### プランA：B2C フリーミアムモデル（旅行者向け）
*無料版ではベーシックな価値を提供し、AIのAPIコストがかかるリッチな体験を課金ポイントにする。*
- **無料ユーザー:** 月X回までのメニューテキスト翻訳のみ。
- **プレミアム会員 (Pro):** 月額 / あるいは「7日間旅行パス」のようなワンタイム課金。
  - 料理のイメージ画像生成（Hero画像・個別料理画像）のアンロック。
  - アレルギー＆ハラール等の詳細な成分推測定。
  - 保存・履歴の無制限アクセス。

### プランB：B2B / B2B2C モデル（飲食店・ホテル向け）
*インバウンド対応に悩む飲食店側にアプローチする。*
- **メニューのスマートデジタル化ツールとして販売:** 飲食店が自分の店の日本語メニューを撮影するだけで、一瞬で「多言語対応＋料理画像付き」のデジタルメニュー（QRコード読み取り式）が生成されるSaaS。
- お店側は月額費用を払い、来店した外国人観光客は自身のスマホでQRを読み取るだけで `menumenu` のリッチなUIでメニューを母国語で読める。初期のメニュー多言語化の翻訳コストや写真撮影コストを大幅に削減できる。

### プランC：アフィリエイト・送客連携
- 旅行先でのレストラン検索として。解析したメニューのレストラン情報や周辺情報から、TripAdvisor、Yelp、OpenTable等の予約サイトや、UberEats等へ誘導し送客手数料を得る。
