**作成日**: 2026-03-22
**目的**: UIデザイナーによるデザインレビューおよび改善提案の依頼

---

## 1. プロダクト概要

### What
**Nomameshi** — 外国語メニューをAIで翻訳・視覚化するモバイルアプリ（iOS / PWA）

### How it works
1. ユーザーがレストランのメニュー写真を撮影 or アップロード
2. AIがメニューを構造化・翻訳（料理名、説明、価格、レストランの雰囲気）
3. AIが料理のテーブルスプレッド画像を自動生成
4. グラスモーフィズムのカードUIで翻訳結果を背景画像の上に表示
5. 結果をPNG画像として保存 or シェア可能

### ターゲットユーザー
- スローノマド（数週間〜数ヶ月単位で海外に滞在しながら働く人）
- 食に関心が高く、観光客向けレストランよりローカル食堂を好む
- 「旅行ツール」ではなく「日常の食事の道具」として使う

## 2. 現在のデザインシステム

### カラーパレット
| Token | 値 | 用途 |
|---|---|---|
| Primary | `#E85A4F` | CTAボタン、アクセント（テラコッタ / コーラル） |
| Secondary | `#8A9178` | サブ要素（ミュートセージグリーン） |
| Accent | `#C9A86C` | セクションタイトル（ゴールド） |
| Background | `#FCFBF9` | ページ背景（ウォームクリーム） |
| Surface | `#FFFFFF` | カード表面 |
| Foreground | `#1E2432` | テキスト（ダークネイビー） |

### タイポグラフィ
- **見出し**: Outfit (Google Fonts), weight 600-700
- **本文**: Inter (Google Fonts)
- letter-spacing: -0.01em〜-0.04em（タイト）

### レイアウト
- モバイルファースト、max-width: 600px
- 角丸: 8px / 16px / 24px の3段階
- Safe area insets対応（ノッチ端末）

### 技術スタック
- Next.js 16 + React 19（静的エクスポート）
- Capacitor（iOS ネイティブブリッジ）
- CSS Variables によるカスタムスタイリング（UIライブラリ不使用）

## 3. 現在の画面構成

### Screen A: ホーム（スキャン待機）
**構成要素（上から順に）:**
1. ロゴ「noma**meshi**」+ サブタイトル "Your local meal companion"
2. リージョン選択ドロップダウン（Auto-Detect / 9カ国）
3. 言語トグル（日本語 / English）
4. スキャンエリア（カメラアイコン + "Scan a menu" + Scan / Upload ボタン）
5. LOCAL TIP セクション（選択リージョンに応じた食文化の豆知識）

### Screen B: 解析結果
**構成要素:**
1. ツールバー（← Back / Share / Save）
2. 背景レイヤー: AI生成テーブルスプレッド画像（9:16縦長、ダークムーディー）
3. グラスモーフィズムのメニューカード:
   - レストラン名 + Vibe（雰囲気の説明）
   - セクション区切り（カテゴリ名: 翻訳 + 原語）
   - 各料理: 翻訳名（太字）/ 原語名（イタリック）/ 価格（赤ピルバッジ）/ 説明文
4. フッター: "Translated by nomameshi"

## 4. 現状の課題と改善してほしいポイント

### 4-1. ホーム画面の印象が弱い
- カメラアイコン + ボタン2つだけのスキャンエリアが**素朴すぎる**
- アプリを開いた瞬間の「おっ」という第一印象が足りない
- ロゴ周りのブランド感が薄い（テキストのみ、装飾なし）
- 全体的に「よくできたMVP」感が残っている

### 4-2. Tipsセクションのビジュアル
- テキストの羅列感が強い
- 国旗 + テキストだけで、カルーセルのインタラクション感が弱い
- ローディング中の「楽しさ」をもっと演出したい

### 4-3. 結果画面のカード可読性
- グラスモーフィズムの透過度と背景画像のコントラストが、AI生成画像の明暗によって不安定
- 暗い背景画像では良いが、明るい食材（パスタ等）の上だと文字が読みにくくなるケースがある
- 価格バッジのデザインが画面内で少し浮いている

### 4-4. ブランドの統一感
- ロゴ、カラー、フォントの「nomameshiらしさ」がまだ確立されていない
- 「ウォーム＆ローカル」の方向性は出ているが、もっとクラフト感・手触り感がほしい
- 保存したPNG画像（結果のスクリーンショット）がSNS映えするレベルに到達していない

### 4-5. インタラクション・マイクロアニメーション
- 画面遷移やボタンのフィードバックが最低限
- スキャン → ローディング → 結果表示のフロー全体で、もっと「ワクワク感」を演出できる余地がある

## 5. デザインの方向性（希望）

### 維持したいもの
- **ウォームトーン**（テラコッタ、アースカラー系）— 前回のリブランディングで定めた方向性
- **モバイルファースト** — これは崩さない
- **グラスモーフィズム** — 結果画面の基本構造は維持（透過度・色味の調整はOK）
- **ノイズレスなキャンバス** — 主役はAI生成の料理画像。UIはそれを邪魔しない

### 変えたい / 探りたいもの
- ホーム画面のレイアウト・ビジュアルヒエラルキー
- ロゴまわりのブランド表現（タイポロゴの洗練 or ミニマルなアイコンの追加）
- Tipsセクションのビジュアル処理
- カラーパレットの微調整（現行をベースに）
- 結果カードの可読性改善
- 全体のクラフト感・仕上げ品質の向上

## 8. 制約・注意事項

- **CSSカスタム変数ベース**のスタイリングなので、デザイントークンの変更は比較的容易
- 外部UIライブラリは未使用。コンポーネントはすべてカスタム実装
- 対応言語: 日本語、英語、中国語、韓国語、スペイン語、フランス語（UIテキストは多言語）
- 結果画面は**html2canvasでPNGエクスポート**されるため、CSSの一部（backdrop-filter等）がエクスポート時に再現されない制約あり
